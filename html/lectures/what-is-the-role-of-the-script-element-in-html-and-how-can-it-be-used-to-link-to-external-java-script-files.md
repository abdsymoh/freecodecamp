# ما هو دور عنصر **`script`** في HTML، وكيف يمكن استخدامه للربط بملفات JavaScript الخارجية؟

-----

يُستخدم **عنصر `script`** لتضمين **كود قابل للتنفيذ**.

يستخدم معظم المطورين هذا العنصر لتنفيذ كود **JavaScript**. تُستخدم لغة JavaScript لإضافة **تفاعلية (Interactivity)** إلى صفحات الويب الخاصة بك. تتضمن الأمثلة الشائعة لاستخدام JavaScript ما يلي: الألعاب التفاعلية، ومُنزلقات الصور (Image sliders)، والنماذج الديناميكية التي تتحقق من صحة مدخلات المستخدم في الوقت الفعلي.

إليك مثال على استخدام عنصر **`script`** داخل مستند HTML:

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

في هذا المثال، لدينا أمر **`alert`** (تنبيه) لعرض رسالة "Welcome to freeCodeCamp". عند تحميل الصفحة لأول مرة، سيظهر هذا التنبيه المنبثق. بعد ذلك، يمكن للمستخدم النقر على زر "موافق" لإغلاق الرسالة.

-----

### الربط بملفات JavaScript الخارجية

على الرغم من أنه يمكنك تقنيًا كتابة كل كود JavaScript الخاص بك داخل وسمَي **`<script>`**، إلا أنه يُعتبر **أفضل الممارسات** هو الربط بملف JavaScript **خارجي** بدلاً من ذلك.

إليك مثال على استخدام عنصر **`script`** للربط بملف JavaScript خارجي:

```html
<script src="path-to-javascript-file.js"></script>
```

تُستخدم **خاصية `src`** هنا لتحديد موقع ملف JavaScript الخارجي هذا. كلمة **`src`** هي اختصار لكلمة **"source"** (المصدر).

-----

### مبدأ فصل الاهتمامات (Separation of Concerns)

السبب وراء عدم تشجيع وضع كل كود JavaScript الخاص بك داخل مستند HTML هو مبدأ **فصل الاهتمامات (Separation of Concerns)**.

**فصل الاهتمامات** هو مبدأ تصميم يتم فيه تقسيم برامجك إلى أقسام متميزة، ويتناول كل قسم اهتمامًا (Concern) منفصلاً. في هذه الحالة، نريد فصل كود **JavaScript** (السلوك/التفاعل) عن كود **HTML** (الهيكل).
